# Validate IBAN (Full)

<Info>**Credits:** 3 per call</Info>

Validates an IBAN number and returns comprehensive details including bank name, BIC/SWIFT code, branch information, SEPA membership, and country data. Supports 110+ countries.

## Request

<ParamField path="iban" type="string" required>
  The IBAN to validate. Can include spaces (they are stripped automatically).
</ParamField>

```bash
GET /services/iban/validate/{iban}
```

Both `GET` and `POST` methods are supported.

<CodeGroup>

```bash cURL
curl -X GET "https://api.edge.bh/api/v1/services/iban/validate/GB29NWBK60161331926819" \
  -H "Authorization: Bearer sk_live_your_key_here"
```

```javascript JavaScript
const response = await fetch(
  "https://api.edge.bh/api/v1/services/iban/validate/GB29NWBK60161331926819",
  { headers: { "Authorization": "Bearer sk_live_your_key_here" } }
);
const data = await response.json();
```

```python Python
import requests

response = requests.get(
    "https://api.edge.bh/api/v1/services/iban/validate/GB29NWBK60161331926819",
    headers={"Authorization": "Bearer sk_live_your_key_here"}
)
data = response.json()
```

</CodeGroup>

## Response

<ResponseField name="valid" type="boolean">
  Whether the IBAN is valid.
</ResponseField>

<ResponseField name="iban" type="string">
  The normalized IBAN (spaces removed, uppercased).
</ResponseField>

<ResponseField name="bank_data" type="object | null">
  Bank and account information. `null` if the bank could not be identified.
  <Expandable>
    <ResponseField name="bic" type="string | null">8-character BIC/SWIFT code.</ResponseField>
    <ResponseField name="bic11" type="string | null">11-character BIC/SWIFT code (with branch suffix).</ResponseField>
    <ResponseField name="bank" type="string | null">Bank name.</ResponseField>
    <ResponseField name="branch" type="string | null">Branch name or identifier.</ResponseField>
    <ResponseField name="address" type="string | null">Bank address.</ResponseField>
    <ResponseField name="city" type="string | null">Bank city.</ResponseField>
    <ResponseField name="state" type="string | null">Bank state/region.</ResponseField>
    <ResponseField name="zip" type="string | null">Bank postal/ZIP code.</ResponseField>
    <ResponseField name="phone" type="string | null">Bank phone number.</ResponseField>
    <ResponseField name="fax" type="string | null">Bank fax number.</ResponseField>
    <ResponseField name="www" type="string | null">Bank website URL.</ResponseField>
    <ResponseField name="email" type="string | null">Bank email address.</ResponseField>
    <ResponseField name="country" type="string | null">Country name.</ResponseField>
    <ResponseField name="country_iso" type="string">ISO 3166-1 alpha-2 country code.</ResponseField>
    <ResponseField name="currency_code" type="string | null">ISO 4217 currency code.</ResponseField>
    <ResponseField name="bban" type="string">Basic Bank Account Number (IBAN without country code and check digits).</ResponseField>
    <ResponseField name="account_number" type="string | null">Account number extracted from the IBAN.</ResponseField>
    <ResponseField name="bank_code" type="string | null">Bank code extracted from the IBAN.</ResponseField>
    <ResponseField name="branch_code" type="string | null">Branch code extracted from the IBAN.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="sepa_data" type="object">
  SEPA (Single Euro Payments Area) status.
  <Expandable>
    <ResponseField name="member" type="boolean">Whether the country is a SEPA member.</ResponseField>
    <ResponseField name="schemes" type="string[]">Supported SEPA schemes. Possible values: `SCT` (Credit Transfer), `SDD` (Direct Debit), `COR1`, `B2B`, `SCC` (Card Clearing).</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="validations" type="object">
  Keyed object of validation checks performed. Each key maps to an object with `code` and `message`.
  <Expandable>
    <ResponseField name="chars" type="object">`code` and `message` — checks for illegal characters.</ResponseField>
    <ResponseField name="country_support" type="object">`code` and `message` — checks if the country supports IBAN.</ResponseField>
    <ResponseField name="length" type="object">`code` and `message` — checks IBAN length matches country spec.</ResponseField>
    <ResponseField name="structure" type="object">`code` and `message` — checks IBAN structure matches country format.</ResponseField>
    <ResponseField name="iban" type="object">`code` and `message` — checks IBAN check digits (mod-97).</ResponseField>
    <ResponseField name="account" type="object">`code` and `message` — checks account number check digit.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="errors" type="array">
  Array of failed validation checks. Each item has `code` and `message`. Empty array when the IBAN is valid.
</ResponseField>
